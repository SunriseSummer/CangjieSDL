# 渲染、坐标与截图排错

## 先看现象

本页处理窗口已创建后的三类问题：画面空白或闪烁，内容模糊/缩放/命中错位，纹理透明与截图文件异常。先把场景缩成一个背景和一个矩形，确认提交顺序；再恢复坐标与资源。每个修复都要求真实窗口或可打开图片，不以编译成功结束。

## 可能原因

空白通常来自漏掉 `endScene`/`present`、在提交后清屏或裁剪未恢复；缩放错位来自逻辑尺寸与像素尺寸混用、窗口缩放后未刷新、度量与绘制字体参数不同；图片异常来自纹理已关闭、目标矩形无效、混合模式错误、截图时机或输出路径。超采样和垂直同步影响不同，不应互相替代。

## 诊断步骤

### 症状一：窗口存在但全黑、闪烁或只有部分帧

先只保留一种明确背景和一个白色矩形，检查一帧严格按 beginScene、draw、endScene、present 执行一次。打印帧计数和 Renderer 驱动名，确认循环仍在运行。下面探针不依赖字体或纹理，若它也不可见，问题在场景顺序、窗口尺寸或显示环境。确认标准是蓝底白框持续稳定可见。

```cangjie role=probe
let r = window.renderer
r.beginScene(Float32(window.width), Float32(window.height), Color.rgb(15, 23, 42))
r.fill(Rect(40.0, 40.0, 160.0, 90.0), Color.rgb(255, 255, 255))
r.endScene()
r.present()
```

恢复内容时用成对的裁剪和视口操作，必要时在帧开始重置。下面修复显式保证 `popClip`，并把提交放在所有绘制之后。确认白框、裁剪内绿块和裁剪外橙块都处于预期位置。

```cangjie role=fix
r.beginScene(width, height, Color.rgb(15, 23, 42))
r.pushClip(Rect(40.0, 40.0, 180.0, 100.0))
r.fill(Rect(20.0, 20.0, 240.0, 140.0), Color.rgb(34, 197, 94))
r.popClip()
r.fillRoundedRect(Rect(260.0, 40.0, 140.0, 70.0), 12.0, Color.rgb(251, 146, 60))
r.endScene()
r.present()
```

### 症状二：画面模糊、窗口缩放后裁切或鼠标点击偏移

先回答“布局用的是逻辑尺寸还是设备像素”，再同时打印窗口逻辑宽高、实际像素尺寸、像素密度和显示缩放，不要只看一个值。确认矩形和事件坐标都使用逻辑空间，场景接收当前逻辑尺寸，窗口缩放后刷新内部宽高。

字体右对齐时比较度量与绘制参数。修复后把窗口移到不同缩放显示器并复测，确认布局比例、文字和命中仍一致；截图像素尺寸可以变化，逻辑关系不应变化。

### 症状三：纹理黑底、图片不见或 BMP 截图无法打开

检查 Texture 是否由当前 Renderer 创建且尚未关闭，源/目标 Rect 是否为正并在范围内，透明图片是否使用合适混合。把图片替换为程序生成的纯色 Surface，比较文件解码和纹理绘制两层。截图在完整帧后触发，打印绝对输出路径。确认 BMP 存在、非空、尺寸可读、SHA-256 已记录，并实际打开检查；仅 `captureBmp` 没抛异常不够。

## 修复方法

先恢复最小场景顺序，再逐项加回裁剪、文字、纹理和截图。坐标问题统一回逻辑空间；高 DPI 资源选择另行处理。资源问题把加载移到循环外并明确嵌套关闭。截图问题使用稳定可写目录和一次性触发标志。每次恢复一种能力都截图或记录查询值，避免一次性加回后无法定位。

## 确认已经修复

窗口至少连续显示十秒、缩放和移动后仍清晰，点击位置与可见按钮一致；纹理透明正确；截图能打开且宽高、哈希与当前文件一致；关闭窗口退出码为 0。再运行计算器和实时游戏的视觉用例，确认修复没有破坏文字、圆角、HUD 或帧循环。

## 避免再次发生

把场景顺序封装成单一 `draw` 入口，裁剪使用局部成对结构，布局函数只接逻辑尺寸，资源只在初始化加载。为 GUI 示例保存真实截图与命令，为无窗口测试保存断言，两类证据不混用。重构渲染器、字体或高 DPI 处理后重拍当前截图，旧哈希自动失效。
